1) BUSINESS CONTEXT AND FUNCTIONAL OBJECTIVE
============================================================
Build a CFPB 1071 data collection system for small business lending operations.

Primary business workflow:
1. Relationship Manager (RM) uses Dashboard to manage borrowers and loans.
2. RM clicks Collect 1071 for a specific loan.
3. System creates a unique request GUID and stores request metadata.
4. System sends email to borrower with secure link: /form1071/{guid}
5. Borrower opens link and sees prefilled loan context + borrower identity fields.
6. Borrower submits CFPB 1071 data fields.
7. Submission is stored; request status is updated to submitted.
8. Dashboard shows collected status and allows view of submitted data.
9. Resend creates a new request row and GUID while old rows are preserved as inactive history.
10. Old inactive links should not accept new submissions.

Compliance/business behavior details:
- 1071 requests are tracked per loan.
- Only one active request per loan at a time.
- Historical requests are retained for auditability.
- Borrower-facing timestamps are Arizona/Phoenix timezone in UI and email text formatting.
